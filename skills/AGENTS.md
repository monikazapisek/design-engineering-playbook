# Agent Instructions for `skills/`

- Follow `skills-guideline.md` when creating or editing skills.
- Every skill must have one clear job.
- Create one folder per skill.
- Every skill folder must contain `SKILL.md`.
- Optional subfolders: `references/`, `examples/`, `scripts/`.
- Do not add per-skill `README.md` by default.
- Do not create empty optional folders unless the user asks for a full scaffold.
- Do not copy private workspace content directly.
- Keep all content public-safe, generic, and reusable.
- Avoid client-specific, personal, or secret data.
- Ask before deleting, renaming, or broadening the scope of a skill.
- Do not edit root `README.md` without explicit user approval.
